# 적대적 감리

적대적 감리는 원고를 칭찬하는 작업이 아닙니다. 이미 좋아 보이는 원고에서 깨지는 지점을 찾는 작업입니다.

## 기본 태도

- `문제 없음`은 결론이지 출발점이 아니다.
- 자동 evidence를 그대로 믿지 않는다.
- 지적은 반드시 근거와 함께 쓴다.
- 확정 이슈와 유보 이슈를 분리한다.

## 3-pass

각 pass는 다음 축을 최소 단위로 봅니다.

- 시간축/달력 연속성
- 숫자/금액 carryover
- 인물/직함 drift
- 시대 검증/고증 축
- replay vs escalation
- 앞뒤 화 브리지
- 캐릭터성/관계 일관성
- 사건 상태 회귀
- 송고 위생
- 독자-facing 표현/문장

### Pass 1. 표면 팩트 감리

넓게 모읍니다.

- 날짜
- 시간
- 돈
- 퍼센트
- 나이
- 직함
- 관계 호칭
- 장소
- 회차 경계 반복
- 송고 위생 후보

### Pass 2. 반례 감리

Pass 1 후보를 깨봅니다.

- 정말 오류인가?
- 플래시백/의도적 반복/리캡일 가능성은 없는가?
- 같은 수치/인물/직함이 다른 곳에서 어떻게 재사용되는가?
- 문제를 세게 말할 만큼 근거가 충분한가?

### Pass 3. 폐쇄 감리

마지막으로 남은 허점을 봅니다.

- 우선순위가 맞는가?
- 송고 차단과 폴리싱을 섞지 않았는가?
- 작가 메모/HTML 엔티티 같은 위생 문제와 핵심 서사 결함을 구분했는가?
- human-facing 보고서가 1장으로 전달 가능한가?

## FAIL 조건

- 회차/줄번호 없는 지적.
- 반례 탐색 없이 단정.
- 자동 후보 파일을 그대로 재인용.
- "대체로 무난"으로 끝남.
- P0/P1/P2가 섞임.
